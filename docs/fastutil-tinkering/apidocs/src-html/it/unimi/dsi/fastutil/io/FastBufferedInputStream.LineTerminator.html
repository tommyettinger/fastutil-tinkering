027/** Lightweight, unsynchronized, aligned input stream buffering class with
028 *  {@linkplain #skip(long) true skipping},
029 *  {@linkplain MeasurableStream measurability},
030 *  {@linkplain RepositionableStream repositionability}
031 *  and {@linkplain #readLine(byte[], int, int, EnumSet) line reading} support.
032 *
033 * <P>This class provides buffering for input streams, but it does so with
034 * purposes and an internal logic that are radically different from the ones
035 * adopted in {@link java.io.BufferedInputStream}. The main features follow.
036 *
037 * <ul>
038 * <li><P>There is no support for marking. All methods are unsychronized.
039 *
040 * <li><P>As an additional feature, this class implements the {@link
041 * RepositionableStream} and {@link MeasurableStream} interfaces.
042 * An instance of this class will try to cast
043 * the underlying byte stream to a {@link RepositionableStream} and to fetch by
044 * reflection the {@link java.nio.channels.FileChannel} underlying the given
045 * output stream, in this order. If either reference can be successfully
046 * fetched, you can use {@link #position(long)} to reposition the stream.
047 * Much in the same way, an instance of this class will try to cast the
048 * the underlying byte stream to a {@link MeasurableStream}, and if this
049 * operation is successful, or if a {@link java.nio.channels.FileChannel} can
050 * be detected, then {@link #position()} and {@link #length()} will work as expected.
051 *
052 *
053 * <li><p>Due to erratic and unpredictable behaviour of {@link InputStream#skip(long)},
054 * which does not correspond to its specification and which Sun refuses to fix
055 * (see <a href="http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6222822">bug 6222822</a>;
056 * don't be fooled by the &ldquo;closed, fixed&rdquo; label),
057 * this class peeks at the underlying stream and if it is {@link System#in} it uses
058 * repeated reads instead of calling {@link InputStream#skip(long)} on the underlying stream; moreover,
059 * skips and reads are tried alternately, so to guarantee that skipping
060 * less bytes than requested can be caused only by reaching the end of file.
061 *
062 * <li><p>This class keeps also track of the number of bytes read so far, so
063 * to be able to implement {@link MeasurableStream#position()}
064 * independently of underlying input stream.
065 *
066 * <li><p>This class has limited support for
067 * {@linkplain #readLine(byte[], int, int, EnumSet) &ldquo;reading a line&rdquo;}
068 * (whatever that means) from the underlying input stream. You can choose the set of
069 * {@linkplain FastBufferedInputStream.LineTerminator line terminators} that
070 * delimit lines.
071 *
072 * </ul>
073 *
074 * <p><strong>Warning:</strong> Since <code>fastutil</code> 6.0.0, this class detects
075 * a implementations of {@link MeasurableStream} instead of subclasses <code>MeasurableInputStream</code> (which is deprecated).
076 *
077 * @since 4.4
078 */
079

124        private static int ensureBufferSize(final int bufferSize) {
125                if (bufferSize <= 0) throw new IllegalArgumentException("Illegal buffer size: " + bufferSize);
126                return bufferSize;

129        /** Creates a new fast buffered input stream by wrapping a given input stream with a given buffer.
130         *
131         * @param is an input stream to wrap.
132         * @param buffer a buffer of positive length.
133         */
134        public FastBufferedInputStream(final InputStream is, final byte[] buffer) {
135                this.is = is;
136                ensureBufferSize(buffer.length);
137                this.buffer = buffer;
138
139                if (is instanceof RepositionableStream) repositionableStream = (RepositionableStream)is;
140                if (is instanceof MeasurableStream) measurableStream = (MeasurableStream)is;
141
142                if (repositionableStream == null) {
143
144                        try {
145                                fileChannel = (FileChannel)(is.getClass().getMethod("getChannel", new Class[] {})).invoke(is, new Object[] {});
146                        }
147                        catch(IllegalAccessException e) {}
148                        catch(IllegalArgumentException e) {}
149                        catch(NoSuchMethodException e) {}
150                        catch(java.lang.reflect.InvocationTargetException e) {}
151                        catch(ClassCastException e) {}
152                }
153        }

172        /** Checks whether no more bytes will be returned.
173         *
174         * <p>This method will refill the internal buffer.
175         *
176         * @return true if there are no characters in the internal buffer and
177         * the underlying reader is exhausted.
178         */
179
180        protected boolean noMoreCharacters() throws IOException {
181                if (avail == 0) {
182                        avail = is.read(buffer);
183                        if (avail <= 0) {
184                                avail = 0;
185                                return true;
186                        }
187                        pos = 0;
188                }
189                return false;
190        }
191
192
193
194        public int read() throws IOException {
195                if (noMoreCharacters()) return -1;
196                avail--;
197                readBytes++;
198                return buffer[pos++] & 0xFF;
199        }

202        public int read(final byte b[], final int offset, final int length) throws IOException {
203                if (length <= avail) {
204                        System.arraycopy(buffer, pos, b, offset, length);
205                        pos += length;
206                        avail -= length;
207                        readBytes += length;
208                        return length;
209                }
210
211                final int head = avail;
212
213                System.arraycopy(buffer, pos, b, offset, head);
214                pos = avail = 0;
215                readBytes += head;
216
217                if (length > buffer.length) {
218                        // We read directly into the destination
219                        final int result = is.read(b, offset + head, length - head);
220                        if (result > 0) readBytes += result;
221                        return result < 0 ? (head == 0 ? -1 : head) : result + head;
222                }
223
224                if (noMoreCharacters()) return head == 0 ? -1 : head;
225
226                final int toRead = Math.min(length - head, avail);
227                readBytes += toRead;
228                System.arraycopy(buffer, 0, b, offset + head, toRead);
229                pos = toRead;
230                avail -= toRead;
231
232                // Note that head >= 0, and necessarily toRead > 0
233                return toRead + head;
234        }

272        /** Reads a line into the given byte-array fragment.
273         *
274         * <P>Reading lines (i.e., characters) out of a byte stream is not always sensible
275         * (methods available to that purpose in old versions of Java have been mercilessly deprecated).
276         * Nonetheless, in several situations, such as when decoding network protocols or headers
277         * known to be ASCII, it is very useful to be able to read a line from a byte stream.
278         *
279         * <p>This method will attempt to read the next line into <code>array</code> starting at <code>off</code>,
280         * reading at most <code>len</code> bytes. The read, however, will be stopped by the end of file or
281         * when meeting a {@linkplain LineTerminator <em>line terminator</em>}. Of course, for this operation
282         * to be sensible the encoding of the text contained in the stream, if any, must not generate spurious
283         * carriage returns or line feeds. Note that the termination detection uses a maximisation
284         * criterion, so if you specify both {@link LineTerminator#CR} and
285         * {@link LineTerminator#CR_LF} meeting a pair CR/LF will consider the whole pair a terminator.
286         *
287         * <p>Terminators are <em>not</em> copied into <em>array</em> or included in the returned count. The
288         * returned integer can be used to check whether the line is complete: if it is smaller than
289         * <code>len</code>, then more bytes might be available, but note that this method (contrarily
290         * to {@link #read(byte[], int, int)}) can legitimately return zero when <code>len</code>
291         * is nonzero just because a terminator was found as the first character. Thus, the intended
292         * usage of this method is to call it on a given array, check whether <code>len</code> bytes
293         * have been read, and if so try again (possibly extending the array) until a number of read bytes
294         * strictly smaller than <code>len</code> (possibly, -1) is returned.
295         *
296         * <p>If you need to guarantee that a full line is read, use the following idiom:
297         * <pre>
298         * int start = off, len;
299         * while((len = fbis.readLine(array, start, array.length - start, terminators)) == array.length - start) {
300         *     start += len;
301         *     array = ByteArrays.grow(array, array.length + 1);
302         * }
303         * </pre>
304         *
305         * <p>At the end of the loop, the line will be placed in <code>array</code> starting at
306         * <code>off</code> (inclusive) and ending at <code>start + Math.max(len, 0)</code> (exclusive).
307         *
308         * @param array byte array where the next line will be stored.
309         * @param off the first byte to use in <code>array</code>.
310         * @param len the maximum number of bytes to read.
311         * @param terminators a set containing the line termination sequences that we want
312         * to consider as valid.
313         * @return the number of bytes actually placed in <code>array</code>, or -1 at end of file.
314         * Note that the returned number will be <code>len</code> if no line termination sequence
315         * specified in <code>terminators</code> has been met before scanning <code>len</code> byte,
316         * and if also we did not meet the end of file.
317         */
318
319        public int readLine(final byte[] array, final int off, final int len, final EnumSet<LineTerminator> terminators) throws IOException {
320                ByteArrays.ensureOffsetLength(array ,off, len);
321                if (len == 0) return 0; // 0-length reads always return 0
322                if (noMoreCharacters()) return -1;
323                int i, k = 0, remaining = len, read = 0; // The number of bytes still to be read
324                for(;;) {
325                        for(i = 0; i < avail && i < remaining && (k = buffer[pos + i]) != '\n' && k != '\r' ; i++);
326                        System.arraycopy(buffer, pos, array, off + read, i);
327                        pos += i;
328                        avail -= i;
329                        read += i;
330                        remaining -= i;
331                        if (remaining == 0) {
332                                readBytes += read;
333                                return read; // We did not stop because of a terminator
334                        }
335
336                        if (avail > 0) { // We met a terminator
337                                if (k == '\n') { // LF first
338                                        pos++;
339                                        avail--;
340                                        if (terminators.contains(LineTerminator.LF)) {
341                                                readBytes += read + 1;
342                                                return read;
343                                        }
344                                        else {
345                                                array[off + read++] = '\n';
346                                                remaining--;
347                                        }
348                                }
349                                else if (k == '\r') { // CR first
350                                        pos++;
351                                        avail--;
352
353                                        if (terminators.contains(LineTerminator.CR_LF)) {
354                                                if (avail > 0) {
355                                                        if (buffer[pos] == '\n') { // CR/LF with LF already in the buffer.
356                                                                pos ++;
357                                                                avail--;
358                                                                readBytes += read + 2;
359                                                                return read;
360                                                        }
361                                                }
362                                                else { // We must search for the LF.
363                                                        if (noMoreCharacters()) {
364                                                                // Not found a matching LF because of end of file, will return CR in buffer if not a terminator
365
366                                                                if (! terminators.contains(LineTerminator.CR)) {
367                                                                        array[off + read++] = '\r';
368                                                                        remaining--;
369                                                                        readBytes += read;
370                                                                }
371                                                                else readBytes += read + 1;
372
373                                                                return read;
374                                                        }
375                                                        if (buffer[0] == '\n') {
376                                                                // Found matching LF, won't return terminators in the buffer
377                                                                pos++;
378                                                                avail--;
379                                                                readBytes += read + 2;
380                                                                return read;
381                                                        }
382                                                }
383                                        }
384
385                                        if (terminators.contains(LineTerminator.CR)) {
386                                                readBytes += read + 1;
387                                                return read;
388                                        }
389
390                                        array[off + read++] = '\r';
391                                        remaining--;
392                                }
393                        }
394                        else if (noMoreCharacters()) {
395                                readBytes += read;
396                                return read;
397                        }
398                }
399        }

403        public void position(long newPosition) throws IOException {
404
405                final long position = readBytes;
406
407                /** Note that this check will succeed also in the case of
408                 * an empty buffer and position == newPosition. This behaviour is
409                 * intentional, as it delays buffering to when it is actually
410                 * necessary and avoids useless class the underlying stream. */
411
412                if (newPosition <= position + avail && newPosition >= position - pos) {
413                        pos += newPosition - position;
414                        avail -= newPosition - position;
415                        readBytes = newPosition;
416                        return;
417                }
418
419                if (repositionableStream != null) repositionableStream.position(newPosition);
420                else if (fileChannel != null) fileChannel.position(newPosition);
421                else throw new UnsupportedOperationException("position() can only be called if the underlying byte stream implements the RepositionableStream interface or if the getChannel() method of the underlying byte stream exists and returns a FileChannel");
422                readBytes = newPosition;
423
424                avail = pos = 0;
425        }
426
427        public long position() throws IOException {
428                return readBytes;
429        }

445        /** Skips the given amount of bytes by repeated reads.
446         *
447         * <strong>Warning</strong>: this method uses destructively the internal buffer.
448         *
449         * @param n the number of bytes to skip.
450         * @return the number of bytes actually skipped.
451         * @see InputStream#skip(long)
452         */
453
454        private long skipByReading(final long n) throws IOException {
455                long toSkip = n;
456                int len;
457                while(toSkip > 0) {
458                        len = is.read(buffer, 0, (int)Math.min(buffer.length, toSkip));
459                        if (len > 0) toSkip -= len;
460                        else break;
461                }
462
463                return n - toSkip;
464        }

467        /** Skips over and discards the given number of bytes of data from this fast buffered input stream.
468         *
469         * <p>As explained in the {@linkplain FastBufferedInputStream class documentation}, the semantics
470         * of {@link InputStream#skip(long)} is fatally flawed. This method provides additional semantics as follows:
471         * it will skip the provided number of bytes, unless the end of file has been reached.
472         *
473         * <p>Additionally, if the underlying input stream is {@link System#in} this method will use
474         * repeated reads instead of invoking {@link InputStream#skip(long)}.
475         *
476         * @param n the number of bytes to skip.
477         * @return the number of bytes actually skipped; it can be smaller than <code>n</code>
478         * only if the end of file has been reached.
479         * @see InputStream#skip(long)
480         */
481
482        public long skip(final long n) throws IOException {
483                if (n <= avail) {
484                        final int m = (int)n;
485                        pos += m;
486                        avail -= m;
487                        readBytes += n;
488                        return n;
489                }
490
491                long toSkip = n - avail, result = 0;
492                avail = 0;
493
494                while (toSkip != 0 && (result = is == System.in ? skipByReading(toSkip) : is.skip(toSkip)) < toSkip) {
495                        if (result == 0) {
496                                if (is.read() == -1) break;
497                                toSkip--;
498                        }
499                        else toSkip -= result;
500                }
501
502                final long t = n - (toSkip - result);
503                readBytes += t;
504                return t;
505        }

508        public int available() throws IOException {
509                return (int)Math.min(is.available() + (long)avail, Integer.MAX_VALUE);
510        }

512        public void close() throws IOException {
513                if (is == null) return;
514                if (is != System.in) is.close();
515                is = null;
516                buffer = null;
517        }

519        /** Resets the internal logic of this fast buffered input stream, clearing the buffer.
520         *
521         * <p>All buffering information is discarded, and the number of bytes read so far
522         * (and thus, also the {@linkplain #position() current position})
523         * is adjusted to reflect this fact.
524         *
525         * <p>This method is mainly useful for re-reading
526         * files that have been overwritten externally.
527         */
528
529        public void flush() {
530                if (is == null) return;
531                readBytes += avail;
532                avail = pos = 0;
533        }

535        /** Resets the internal logic of this fast buffered input stream.
536         *
537         * @deprecated As of <code>fastutil</code> 5.0.4, replaced by {@link #flush()}. The old
538         * semantics of this method does not contradict {@link InputStream}'s contract, as
539         * the semantics of {@link #reset()} is undefined if {@link InputStream#markSupported()}
540         * returns false. On the other hand, the name was really a poor choice.

544                flush();
